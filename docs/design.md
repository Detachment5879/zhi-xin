# 设计方案（KSTAR 低本版）

## 1. 问题背景

高校教学中存在"千人一面"的资源问题：
- 同一门课程所有学生使用相同的课件和习题
- 学生学习进度和掌握程度不同，但获取的资源相同
- 教师制作个性化资源的成本高、效率低

## 2. 解决方案

基于 **KSTAR 学习循环** 与 **多智能体协同** 技术，设计一套低成本个性化学习资源生成系统：

**核心理念：** 将 KSTAR 循环的"理念全盘吸收"，但在技术落地时加入"熔断机制"和"成本截断"。

### 2.1 KSTAR 循环

```
K（已知）→ 前置测试，诊断学生已有知识水平
S（现状）→ 分析学生当前状态与目标的差距
T（目标）→ 确定当前要攻克的原子级知识点
A（行动）→ AI生成学习资源 → 人类自主学习 → 验收
R（反思）→ 后测评估 → 判断是否达标 → 决定下一步
```

### 2.2 成本截断策略

| 策略 | 具体做法 |
|------|---------|
| 题库本地化 | 出题环节由本地代码完成，不消耗API |
| 诊断降级 | 正确率 ≥ 80% 直接跳过，零成本通过 |
| 熔断机制 | 每知识点最多循环2次，达标阈值80% |
| K更新降级 | 每次只输出一句话更新K，不重写全量报告 |
| 原子粒度 | T是原子级知识点（如"FOB与CIF的区别"），每次生成内容极少 |

## 4. 数据流设计

### 4.1 核心数据结构

```python
# 知识点状态（K 的核心）
class KnowledgeState:
    knowledge_point_id: str       # 知识点ID
    knowledge_point_name: str     # 知识点名称
    status: str                   # 未学习/学习中/已掌握/已跳过
    correct_rate: float           # 最新正确率
    cycle_count: int              # 已循环次数（上限2）
    latest_summary: str           # 最新的一句话状态更新
    history: List[dict]           # 答题历史 [{round, correct_rate, answers}]
    last_updated: datetime

# 题库题目
class Question:
    id: str
    knowledge_point_id: str
    text: str                     # 题目内容
    options: dict                 # 选项 {"A": "...", "B": "...", ...}
    correct_answer: str           # 正确答案
    question_type: str            # 单选/多选/判断
    difficulty: str               # 基础/进阶/挑战
    source: str                   # 本地导入/大模型生成
```

### 4.2 K 状态更新流程

```
R阶段结束 → 反思Agent输出一句话:
  "学生已掌握关税分类，信用证流程仍薄弱（正确率60%），建议下轮加强。"

→ Python脚本解析关键词:
  "已掌握" → status = "已掌握"
  "仍薄弱" → 标记薄弱点
  "正确率60%" → 更新 correct_rate

→ 写入本地KnowledgeState存储
→ 不调用大模型进行二次处理
```

## 5. 成本分析

### 5.1 单轮循环成本

| 阶段 | API调用次数 | 单价(GPT-4o-mini) | 成本 |
|------|------------|-------------------|------|
| K诊断 | 1 | $0.00015/1K input | ~$0.0001 |
| S+T | 1 | $0.00015/1K input | ~$0.0003 |
| A生成 | 1-2 | $0.00015/1K input + $0.0006/1K output | ~$0.001 |
| R评估 | 1 | $0.00015/1K input | ~$0.0001 |
| **单轮合计** | **4-5次** | | **~$0.0015** |

### 5.2 每知识点封顶成本

| 场景 | 轮数 | 总API调用 | 总成本 |
|------|------|----------|--------|
| 学生一次通过 | 1 | 4-5 | ~$0.0015 |
| 学生需再学一轮 | 2 | 8-10 | ~$0.003 |
| 学生2轮不通过(熔断) | 2 | 8-10 | ~$0.003 |
| K阶段已达80% | 0(跳过) | 0 | $0 |

### 5.3 整门课程成本估算

以一门50个知识点的课程为例：

| 学生类型 | 平均每人轮数 | 总API调用 | 总成本 |
|---------|------------|----------|--------|
| 优等生(30%跳过+70%一次过) | 0.7轮 | 350次 | ~$0.05 |
| 中等生(10%跳过+80%一次过+10%两轮) | 1.0轮 | 500次 | ~$0.08 |
| 后进生(5%跳过+50%一次过+45%两轮) | 1.4轮 | 700次 | ~$0.11 |

**使用 Groq 免费API: 全部零成本**

## 6. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 题库题量不足 | 前测/后测缺乏区分度 | 运行时自动补充+教师可随时导入 |
| 大模型生成不准确 | 讲义内容有错 | 质量审核Agent把关 |
| 学生刷题背答案 | 评估失真 | 大模型在A阶段生成变体题 |
| 熔断后学生无人帮助 | 学习卡住 | 记录建议求助清单，反馈给教师 |
| 知识点粒度不一致 | 有的太大有的太小 | 差距分析Agent统一粒度控制 |
