# 🔄 资源生成流程图（KSTAR 学习循环）

## 一、完整流程

```
┌───────────────────────────────────────────────────────────────────┐
│                        课程准备阶段（一次性）                        │
│                                                                   │
│  教师导入已有题库 ──→ 系统自动按知识点分类 ──→ 存入本地SQLite      │
│  支持格式: Excel / CSV / JSON                                     │
│  每个知识点至少3道题（不足时系统会在运行中自动补充）                  │
└───────────────────────────────────────────────────────────────────┘

                                  │
                                  ▼

┌───────────────────────────────────────────────────────────────────┐
│                    KSTAR 学习循环（每知识点运行一轮）                 │
├───────────────────────────────────────────────────────────────────┤
│                                                                   │
│ ┌──── 1. K 阶段（诊断已知） ──────────────────────────────────┐   │
│ │                                                              │   │
│ │  系统: 从题库中抽5题（覆盖当前知识点的不同难度）                │   │
│ │  学生: 在线作答                                              │   │
│ │  本地: 统计正确率 → 按知识点统计对错                          │   │
│ │                                                              │   │
│ │  if 正确率 ≥ 80%:                                            │   │
│ │    → 直接标记"已掌握"，不调用任何Agent，进入下一个知识点        │   │
│ │     （一道API都不花）                                          │   │
│ │  else:                                                        │   │
│ │    → 调用诊断评估Agent(1次API) → 生成诊断报告                 │   │
│ │                                                              │   │
│ └──────────────────────────────────────────────────────────────┘   │
│                              │                                     │
│                              ▼                                     │
│ ┌──── 2. S+T 阶段（确认目标） ───────────────────────────────┐   │
│ │                                                              │   │
│ │  输入: 诊断报告（薄弱点、已掌握点）                            │   │
│ │  Agent: 差距分析Agent(1次API)                                │   │
│ │  输出: 一个原子级T（如"掌握FOB与CIF的区别"）                  │   │
│ │                                                              │   │
│ └──────────────────────────────────────────────────────────────┘   │
│                              │                                     │
│                              ▼                                     │
│ ┌──── 3. A 阶段（行动） ─────────────────────────────────────┐   │
│ │                                                              │   │
│ │  输入: 原子级T                                               │   │
│ │  Agent: 资源生成Agent(1-2次API)                              │   │
│ │  输出: 学习材料(半页讲义) + 3-5道练习题                        │   │
│ │                                                              │   │
│ │  ────────── 人类行动（0 API消耗） ──────────                   │   │
│ │  学生: 阅读学习材料 → 完成练习题                                │   │
│ │  系统: 等待状态（不调用任何API）                                │   │
│ │  学生完成练习后点击"我学完了，开始验收"                          │   │
│ │  ─────────────────────────────────────────────                  │   │
│ │                                                              │   │
│ └──────────────────────────────────────────────────────────────┘   │
│                              │                                     │
│                              ▼                                     │
│ ┌──── 4. R 阶段（反思） ─────────────────────────────────────┐   │
│ │                                                              │   │
│ │  系统: 从题库抽3题（与K阶段不同题）                            │   │
│ │  学生: 作答                                                  │   │
│ │  本地: 统计正确率                                             │   │
│ │                                                              │   │
│ │  Agent: 反思评估Agent(1次API)                                │   │
│ │  → 判断是否达标 + 输出一句话状态更新                           │   │
│ │                                                              │   │
│ └──────────────────────────────────────────────────────────────┘   │
│                              │                                     │
│                              ▼                                     │
│ ┌──── 5. 熔断决策 ───────────────────────────────────────────┐   │
│ │                                                              │   │
│ │  正确率 ≥ 80%? ────是──→ ✅ 知识点通过 → 下一个知识点         │   │
│ │      │                                                     │   │
│ │      否 ──→ 循环次数 = 1? ──是──→ K = K(旧)+一句状态更新       │   │
│ │      │                  │        → 回到S+T阶段，再学一轮      │   │
│ │      │                  否                                   │   │
│ │      │                  │                                    │   │
│ │      │                  ▼                                    │   │
│ │      │         ⛔ 熔断触发 → 标记"建议求助"                    │   │
│ │      │         系统建议向人类老师求助                          │   │
│ │      │         该知识点标记为"已跳过"                          │   │
│ │      │         进入下一个知识点                                │   │
│ │                                                              │   │
│ └──────────────────────────────────────────────────────────────┘   │
│                                                                   │
└───────────────────────────────────────────────────────────────────┘

                                  │
                                  ▼

┌───────────────────────────────────────────────────────────────────┐
│                       课程学习完成                                  │
│                                                                   │
│  最终产出:                                                         │
│  ├ 完整学习报告（已完成/已跳过/建议求助的知识点清单）               │
│  ├ 每个已掌握知识点的学习材料存档                                   │
│  └ 薄弱点总结（给教师的反馈）                                       │
└───────────────────────────────────────────────────────────────────┘
```

## 二、题库管理流程

```
┌──────────────────────────────────────────────────────────────────┐
│                        题库管理                                     │
├──────────────────────────────────────────────────────────────────┤
│                                                                    │
│  初始化:                                                            │
│  教师上传题库文件(Excel/CSV)                                        │
│  → 系统解析列映射（知识点、题目、选项、答案）                         │
│  → 按知识点自动分类存储                                             │
│  → 统计: 每个知识点XX题，共XX题                                     │
│                                                                    │
│  运行时补充:                                                         │
│  当某个知识点本地题量 < 3道时:                                       │
│  → 调用大模型生成补充题（1次API生成3-5道）                          │
│  → 审核后自动入库                                                   │
│  → 后续该知识点出题优先用补充题                                      │
│                                                                    │
│  维护:                                                              │
│  教师可以随时增删改题目                                              │
│  支持按知识点/难度/题型筛选查看                                      │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

## 三、成本控制概览

| 环节 | API调用 | 本地逻辑 | 说明 |
|------|---------|---------|------|
| K阶段出题 | 0 | 题库随机抽 | 学生答对率≥80%时直接跳过，零成本 |
| K阶段诊断 | 0-1 | 统计正确率 | 仅当答对率<80%时调用 |
| S+T阶段 | 1 | 0 | 必须调用，确定目标 |
| A阶段生成 | 1-2 | 0 | 生成学习材料 |
| A阶段人类学习 | 0 | 等待 | 学生离线学习，零成本 |
| R阶段出题 | 0 | 题库随机抽 | 零成本 |
| R阶段评估 | 1 | 统计正确率 | 判断是否达标 |
| K更新 | 0 | Python脚本处理 | 反思Agent的一句话直接入库 |

**每轮封顶: 4-5次API调用，每知识点最多2轮 = 最多10次API**
